FXF Rating Re-imagine Phase 1

Role: Developer

Environment: Java, MySQL, Spring MVC, JSP, Apache Tomcat

Client: FedEx

Duration: 1 year

Domain: Logistics

Team Size: 12

Project Description:

  • This project consists of three applications: Rating Rules Orchestrator, Rating Path Orchestrator, and Legacy Adopter.
  • The Rating Rules Orchestrator has a validator PGA and accepts incoming requests from the FXF gateway. It calls several services (Base rate, Dynamic fuel surcharge, DS) through RESTful web services and receives the response.
  • If the response is ratable, it sends it to the Rating Path. If it is not ratable, it sends it to Legacy for further processing.
  • The Legacy Adopter calls the E4YC Model and acts as a validator. It is responsible for converting the model to JSON and JSON to model conventions. The response is then sent back to the Rating Path for further processing.

Roles and Responsibilities:

  • Developed a Rating Rules Orchestrator application, using a Controller to handle HTTP POST requests at /api/rating and route them to the service layer.
  • Implemented a Service that validates requests, calls external RESTful services using Rest Template, and processes responses. Integrated Rest Template to fetch data from services like Base Rate and Dynamic Fuel Surcharge.
  • Implemented Response Processing to forward ratable responses to the Rating Path and non-ratable responses to the Legacy Adopter.
  • Configured Rest Template and implemented basic request validation within the Spring Boot application.
